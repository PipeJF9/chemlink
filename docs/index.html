Why Chemlink

Purpose-built for HPC

Running distributed molecular simulations on HPC infrastructure is complex, fragile, and deeply manual. Chemlink abstracts the cluster orchestration layer so your team can focus on chemistry, not plumbing.

01

Unified CLI

One command interface for job submission, monitoring, data collection, and post-processing across SLURM, PBS, and bare-metal clusters.

02

Simulation agnostic

Works with GROMACS, NAMD, LAMMPS, and custom engines through a pluggable backend adapter system.

03

Distributed by design

Native MPI coordination, automatic work partitioning, and fault-tolerant checkpointing built into every task runner.

04

Result aggregation

Collect, normalize, and export trajectory data from all nodes as HDF5 or CSV. Plug directly into your analysis pipeline.

05

Reproducible runs

Lock-file based environments ensure simulations are fully reproducible across clusters, users, and time.

06

Extensible

Write custom plugins in Python to extend any stage of the pipeline — from input generation to analysis and reporting.

Configuration

Chemlink uses a chemlink.toml at the project root, or ~/.chemlink/config.toml for global settings.

Minimal config
toml# chemlink.toml
[cluster]
scheduler = "slurm"
nodes     = 4
cores_per_node = 32

[simulation]
backend    = "gromacs"
input_dir  = "./inputs"
output_dir = "./results"

[mpi]
enabled        = true
ranks_per_node = 8
Environment variables
VariableDescription

CHEMLINK_BACKENDOverride simulation backend
CHEMLINK_LOG_LEVELdebug | info | warn | error
CHEMLINK_MPI_CMDCustom mpirun / mpiexec path

Example workflow
bash# 1. Initialize
chemlink init myproject && cd myproject

# 2. Run distributed simulation
chemlink run --nodes 8 --time 04:00:00 inputs/protein.gro

# 3. Monitor
chemlink status --watch

# 4. Collect & export
chemlink collect --job-id XXXX
chemlink export  --format hdf5 --out results/traj.h5

Plugin API

Chemlink plugins are Python packages implementing one or more plugin interfaces. They are discovered automatically when installed in the same environment.

Plugin structure
python# myplugin/__init__.py
from chemlink.plugins import BackendPlugin

class MyEnginePlugin(BackendPlugin):
    name = "my-engine"

    def prepare(self, config):
        ...   # generate input files

    def run(self, job):
        ...   # submit to scheduler

    def collect(self, job):
        ...   # gather output trajectories
Register entry point
toml# pyproject.toml
[project.entry-points."chemlink.backends"]
my-engine = "myplugin:MyEnginePlugin"
Available hooks
  • BackendPlugin — custom simulation engines
  • AnalysisPlugin — post-processing pipelines
  • ExporterPlugin — custom output formats
  • SchedulerPlugin — custom job schedulers
HPC cluster guide
SLURM setup

Set scheduler = "slurm" in your config. Chemlink generates and submits SBATCH scripts automatically.

bashchemlink run inputs/system.gro \
  --nodes 16 --partition gpu \
  --time 12:00:00 \
  --slurm-extra "--gres=gpu:4"
PBS / Torque
toml[cluster]
scheduler = "pbs"
queue     = "workq"
Bare metal / SSH

For clusters without a scheduler, use scheduler = "bare". Chemlink will SSH into each node and launch MPI workers directly.

Performance tips
  • Set ranks_per_node to match physical core count
  • Use --checkpoint for runs longer than 4 hours
  • Set CHEMLINK_LOG_LEVEL=warn for production
  • Run chemlink audit to validate config before submission
